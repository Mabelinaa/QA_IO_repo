Diseño (NUnit)

Output: Test Suite, Test cases

Estructura básica del proyecto​

Microsoft.Playwright.NUnit proporciona la interfaz PageTest para implementar en los archivos de prueba. Esta interfaz permite inicializar el navegador con una configuración básica.

Cada bloque de código decorado con el atributo [test] aparece en el explorador de pruebas para ser ejecutado. El explorador de pruebas de Visual Studio también muestra información sobre el resultado de las pruebas.

Configuración del proyecto​

La configuración por defecto de PageTest no incluye funcionalidades como capturas de pantalla, grabación de vídeo o generación de reportes. Para personalizar la configuración, o bien se sobreescriben los métodos Browser y Page, o bien se realiza la configuración global a través del archivo runsettings file.

  • Override (Sobrescritura)
  • Runsettings File

En Playwright NUnit se puede customizar el contexto de las pruebas a través de la sobreescritura de ContextOptions (línea 59) y a través de la instanciación de las clases Browser y Page (líneas 8, 9).

using Microsoft.Playwright.NUnit;
using Microsoft.Playwright;

namespace PlaywrightTestPlanWithDotNet
{
    public class Tests : PageTest
    {
        private IBrowser _browser;
        private IPage _page;

        [SetUp]
        public async Task Setup()
        {
            // Inicializar el navegador en modo de visualización
            _browser = await Playwright.Chromium.LaunchAsync(new BrowserTypeLaunchOptions
            {
                Headless = false
            });

            // Configuración global para el contexto del navegador
            var contextOptions = ContextOptions();
            var context = await _browser.NewContextAsync(contextOptions);

            // Crear una nueva página en el contexto del navegador
            _page = await context.NewPageAsync();

            // Extra: trazabilidad de las pruebas
            await context.Tracing.StartAsync(new()
            {
                Screenshots = true,
                Snapshots = true,
                Sources = true
            });
        }

        [Test]
        public async Task Test1()
        {
            await _page.GotoAsync("");
            await _page.GetByText("GET STARTED").ClickAsync();

            //Assert
            Assert.Pass();
        }

        [TearDown]
        public async Task TearDown()
        {
            // Extra: Registro de archivo de trazabilidad
            await _page.Context.Tracing.StopAsync(new()
            {
                Path = "trace.zip"
            });
            // Cierre del navegador
            await _browser.CloseAsync();
        }

        // Extra: Sobrescritura de ContextOptions
        public override BrowserNewContextOptions ContextOptions()
        {
            return new BrowserNewContextOptions()
            {
                // Extra: Grabación de video
                RecordVideoDir = "video",
                // Extra: Variable URL
                BaseURL = "https://playwright.dev/",
            };
        }
    }
}

Los archivos generados (capturas, vídeos, reportes) se registran en \NUnitProject\bin\Debug\netx\.

La trazabilidad de las pruebas (lin 28) se inicia para generar un reporte HTML al que se accede a través del siguiente comando.

pwsh bin/Debug/netX/playwright.ps1 show-trace https://example.com/trace.zip

Este archivo sin extensión se crea en el directorio principal y permite realizar algunas personalizaciones como:

  • <Playwright>: Navegador de pruebas <BrowserName>, Tiempos de espera <ExpectTimeout>, Ejecución en modo ventana <Headless>false</Headless> ó velocidad de reproducción <SlowMo>.
  • <TestRunParameters>: Pasar como parámetros variables a través de la etiqueta <TestRunParameters><Parameter/></TestRunParameters>.
<?xml version="1.0" encoding="utf-8"?>
  <RunSettings>
    <TestRunParameters>
     <Parameter name="BaseUrl" value="https://playwright.dev/" />
    </TestRunParameters>
    <NUnit>
      <NumberOfTestWorkers>24</NumberOfTestWorkers>
    </NUnit>
    <!-- General run configuration -->
    <RunConfiguration>
      <EnvironmentVariables>
        <!-- For debugging selectors, it's recommend to set the following environment variable -->
        <DEBUG>pw:api</DEBUG>
      </EnvironmentVariables>
    </RunConfiguration>
    <!-- Playwright -->  
    <Playwright>
      <BrowserName>chromium</BrowserName>
      <ExpectTimeout>5000</ExpectTimeout>
      <LaunchOptions>
        <Headless>false</Headless>
        <Channel>msedge</Channel>
      </LaunchOptions>
    </Playwright>
  </RunSettings>

Si se estalecen parámetros globales estos pueden ser consumidos en los tests de la siguiente manera:

  namespace PlaywrightTestPlanWithDotNet
  {
      public class UnitTestWithRunSettings : PageTest
      {
          [SetUp]
          public async Task Setup()
          {
              //Arrange (usando parámetro "BaseUrl")
              var baseUrl = TestContext.Parameters["BaseUrl"]
                  ?? throw new Exception("WebAppUrl no está configurada como parámetro");
              await Page.GotoAsync(baseUrl);
          }
          [Test]
          public async Task Test1()
          {
              //Act
              await Page.GetByText("GET STARTED").ClickAsync();
          }
      }
  }

Para que las configuraciones del archivo runsettings funcionen correctamente hay que asgurarse de que se copie siempre en la carpeta /bin del proyecto, restaurarlo y añadir el archivo a las propiedades del proyecto.

POM: Page object models​

POM es un enfoque de organización del proyecto que consiste en crear Page Object para cada una de las páginas o componentes de la interfaz de usuario, y encapsular en cada POM los elementos de la interfaz y las acciones que se pueden realizar en esa página. Los archivos POM se almacenan en el directorio pages y los archivos de pruebas en el directorio tests.

pages: Cada pantalla de la UI corresponde con un archivo en el que se crean los objetos y los métodos que serán invocados desde los tests.
BasePageTest.cs: Plantilla de configuración del contexto

HomePage.cs: Corresponde a la "Home" de la web a probar

tests: archivos de prueba para cada set de pruebas dónde se consumirán los objetos y métodos de pages.
LoginTest.cs: Conjunto de pruebas de la funcionalidad login

El archivo de pruebas LoginTest se crea con la interfaz BasePageTest para acceder a la configuración personalizada y se instancia la clase HomePage para acceder a los objetos y métodos.

Ejecución de las pruebas​

Se pueden ejecutar las pruebas desde la terminal cli o desde el explorador de pruebas de Visual Studio.

  • CLI
  • Test Explorer
  • Ejecutar todos los tests
  PS C:\..\PlaywrightTestPlanWithDotNet> dotnet test
  • Ejecutar un test por nombre
  PS C:\..\PlaywrightTestPlanWithDotNet> dotnet test --filter "Name~LoginFaro"
  • Ejecutar test en un navegador específico
  PS C:\..\PlaywrightTestPlanWithDotNet> dotnet test -- Playwright.BrowserName=firefox

Resultados de las pruebas

Desde el explorador de pruebas de Visual Studio se pueden ejecutar los tests de manera indivudual o conjuntamente. En la pantalla derecha se muestran los resultados de los tests ejecutados: logs, mensajes de error, duración del test...

Generador de pruebas​

Para generar tests a través de la grabadora de Playwright se ejecuta el siguiente comando en la consola:

  PS C:\..\PlaywrightTestPlanWithDotNet> pwsh bin/Debug/netX/playwright.ps1 codegen

Se puede iniciar la grabadora en una url específica indicando en el comando la dirección web.

 PS C:\..\PlaywrightTestPlanWithDotNet> pwsh bin/Debug/netX/playwright.ps1 codegen https://www.google.es/

Con la grabadora se genera el código automáticamente y se puede copiar a los tests creados en el lenguaje seleccionado.
