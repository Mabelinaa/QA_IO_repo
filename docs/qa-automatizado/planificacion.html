Fase de Planificación

Output: Plan de Pruebas

1. Creación de un Plan de Pruebas en el menú Test Plans de Azure Devops.​

Este paso es necesario para vincular los casos de prueba desarrollados en Visual Studio y los casos de prueba creados en Azure Test Plans. El nombre del Plan de Pruebas corresponde con el sprint o entregable a cliente final.

2. Repositorio en Azure Devops​

En caso de no existir un repositorio de pruebas en el proyecto, independiente del código fuente de la aplicación a probar, se crea un repositorio para registrar en él todos los planes de pruebas.

Cada carpeta dentro del repositorio corresponde a un plan de pruebas diferente.

3. Creación del proyecto​

Tras clonar el repositorio en el equipo local, se crea el proyecto en Visual Studio usando como lenguaje .Net y como framework NUnit. Existen otros frameworks válidos para el desarrollo de pruebas, como XUnit o MSTest pero, siguiendo las recomendaciones de la documentación oficial, NUnit es el framework más idóneo para los proyectos de pruebas.

dotnet new nunit -n PlaywrightTestPlanWithDotNet
cd PlaywrightTestPlanWithDotNet
- Añadir dependencias de Playwright al proyecto​
dotnet add package Microsoft.Playwright.NUnit
dotnet build
- Instalación de navegadores de Playwright para ejecución de pruebas​
pwsh bin/Debug/net7.0/playwright.ps1 install
warning

Requerido PowerShell para instalación de Playwright. Link de instalación.
